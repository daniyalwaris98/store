## Design Context

### Users

- **Who**: mobile-first shoppers, predominantly ages 16+, coming from TikTok and social media
- **Context**: Browsing on mobile during commutes, breaks, or casual browsing sessions — no intent, just discovery
- **Job to be done**: Discover products, evaluate quickly, purchase with minimal friction — all on mobile
- **Mindset**: Impulse-driven, value-conscious but aspirational — they want quality AND deals

### Brand Personality

**Bold · Premium · Trusted**

- **Voice**: Clean and confident — no fluff, no gimmicks. Every word earns its place.
- **Tone**: Professional and reassuring on product pages. Urgent and exciting near checkout/sales.
- **Emotional Goals**: Users should feel they're buying from a serious, legitimate store — not a sketchy reskin. Luxury signal without being intimidating.

### Aesthetic Direction

**Visual Tone**: Shopify-meets-Stripe — minimal, surgical, high-trust. Black-and-white dominant with a single blue accent. Clean like Stripe's dashboard, shoppable like Shopify.

**Key Design Characteristics** (derived from references):

- **Dominant palette**: White background, near-black text, `#2563eb` (blue) as the single brand accent — used sparingly for CTAs, links, and active states only
- **Cards**: White surfaces, `rounded-xl` (16px), subtle borders, soft shadows that elevate on hover
- **Typography**: Inter font, tight leading, strong hierarchy. Hero text up to `text-4xl font-bold`. Section headings `text-xl`. Body `text-sm` to `text-base`.
- **White space**: Generous on hero and product pages. Denser on collection/admin grids — but never cramped.
- **Gradients**: Subtle on skeletons only. Primary gradient on buttons for depth.
- **Data presentation**: Compact like Stripe — clean tables, small stat badges, readable numbers.
- **Icons**: Lucide React — consistent stroke weight, used as anchors not decorations
- **Buttons**: Rectangular (`rounded-lg`) for all actions. Filled primary, outlined secondary. Lifts + shadows on hover.
- **Imagery**: Product-first. No decorative illustrations. White/off-white product photo backgrounds preferred.

**Anti-References**:

- AI slop websites — generic templates, cartoon illustrations, overuse of gradients and "modern UI" clichés
- Overly decorative or busy interfaces
- Generic healthcare blue-and-white templates
- Instagram-eque busy layouts

**Theme**: Light mode only. No dark mode support needed.

### Design Principles

1. **Stripe-Level Clarity** — Every element has one job. No decorative chrome. No ambiguity in hierarchy. If it doesn't help the user buy, remove it.

2. **Mobile-First, Always** — 99%+ of users are on mobile. Design at 375px first. Touch targets 44px+. Cart/checkout must be thumb-friendly.

3. **Trust Through Restraint** — Black and white signals seriousness. Minimal color signals confidence. Use blue only where it earns its place — CTAs, active states, brand moments.

4. **Fast to Browse, Fast to Buy** — Product discovery must be frictionless. Checkout must be Stripe-simple. Every step removed is money saved.

5. **Card-Driven but Not Card-Dependent** — Use cards for product grids and summaries. But don't over-card every section. Hero, collection headers, and checkout can break the grid intentionally.

6. **Elevation via Shadows** — Components gain shadow on hover, creating a subtle lift effect. Never flat-lay everything.

7. **Subtle Animations** — 150–300ms transitions with ease-out. Smooth but not showy.

## Shadow System (Elevation)

Components use shadows to communicate depth and interactivity. Shadows escalate on hover.

| Token | Effect | Usage |
|---|---|---|
| `--shadow-xs` | Subtle lift | Small elements, badges |
| `--shadow-sm` | Default state | Buttons, cards at rest |
| `--shadow-md` | Hover state | Cards, buttons on hover |
| `--shadow-lg` | Elevated | Dropdowns, popovers |
| `--shadow-xl` | Maximum | Dialogs, sheets, modals |

## Transitions

| Token | Duration | Easing | Usage |
|---|---|---|---|
| `--transition-fast` | 150ms | ease | Micro-interactions |
| `--transition-base` | 200ms | ease | Default, natural deceleration |
| `--transition-slow` | 300ms | ease | Larger elements, modals |
| `--transition-spring` | 300ms | cubic-bezier(0.34, 1.56, 0.64, 1) | Bouncy, playful feedback |

## Hover Behavior Patterns

All interactive components follow consistent patterns:

| Pattern | Implementation |
|---|---|
| **Lift** | `-translate-y-0.5` on hover |
| **Shadow escalation** | `shadow-sm` → `shadow-md` on hover |
| **Border strengthening** | `border-border` → `border-border-strong` on hover |
| **Easing** | `ease-out` for natural deceleration |

## Component Treatment

### Buttons

- `rounded-lg` with `h-11 px-5 py-2.5` padding
- Default: Primary fill, lifts + shadow on hover
- Destructive: Red fill (`--danger`), lifts + shadow on hover
- Outline: Border-2, subtle background on hover
- Secondary: Muted fill, subtle lift on hover
- Ghost: No background, subtle hover state
- Link: Underline on hover

### Cards

- `rounded-xl` with `shadow-sm` at rest
- Hover: Lifts `-translate-y-0.5`, gains `shadow-md`, border strengthens to `border-border-strong`
- Transition: `all duration-200 ease-out`

### Dialog / Sheet Overlay

- Frosted glass effect: `bg-black/60 backdrop-blur-sm`
- Content: `rounded-xl` with `shadow-xl`
- Close button: `rounded-lg`, hover opacity + background transition

### Dropdown Menu

- Content: `rounded-xl`, `shadow-lg`, `p-1.5` padding
- Items: `rounded-lg`, `py-2.5`, hover `focus:bg-accent-light`
- Sub-trigger: Same rounded treatment

### Select

- Trigger: `rounded-lg`, `h-11`, hover `border-border-strong`
- Content: `rounded-xl`, `shadow-lg`
- Items: `rounded-lg`, `py-2.5`, hover `focus:bg-accent-light`

### Tabs

- List: `rounded-xl`, `bg-background-muted`, `p-1.5`, `shadow-sm`
- Trigger: `rounded-lg`, `px-5 py-2.5`, active `font-semibold` + `shadow-sm`
- Content: `mt-4` margin

### Input / Textarea

- `rounded-lg`, `h-11` height for inputs
- Hover: `border-border-strong`
- Focus: `ring-2 ring-accent ring-offset-2`
- Textarea: `min-h-[120px]`, `resize-none`

### Table

- Container: `rounded-xl`, `border-border`, `shadow-sm`
- Rows: Hover `bg-background-muted`, transition on all
- Header: `font-semibold`
- Footer: `bg-background-muted`

### Toast

- Variants use `rounded-xl`, `p-5` padding
- Action/close: `rounded-lg`, transition on all states

## Typography

- **Font Family**: `Inter` (Google Fonts) — clean, modern, excellent readability at all sizes
- **Hero Headings**: `text-2xl` to `text-4xl`, `font-bold`
- **Section Headings**: `text-base` to `text-lg`, `font-semibold`
- **Card Titles**: `text-xs` to `text-sm`, `font-medium` to `font-semibold`
- **Body Text**: `text-xs` to `text-sm`, regular weight, `text-secondary` color
- **Labels/Captions**: `text-xs`, `font-medium`, `text-secondary`
- **Prices**: `text-sm` to `text-base`, `font-bold`, always visible (no hidden pricing)

## Spacing & Layout

- **Max Width**: `1280px` (container), `768px` for form-focused layouts
- **Section Gaps**: `gap-4` to `gap-6` (tight like Shopify/Stripe)
- **Card Padding**: `p-4` to `p-5` (16–20px)
- **Card Border Radius**: `rounded-xl` (16px)
- **Grid Gaps**: `gap-3` to `gap-4` (12–16px)
- **Mobile Gaps**: `gap-2` (8px) — ultra-dense mobile grids

## Component Library

- **Shadcn UI** (`@radix-ui/react-*` based) — primary component library
- **Tailwind CSS v4** — utility-first styling alongside Shadcn
- **Lucide React** — icon library
- **Radix Primitives**: Dialog, DropdownMenu, Select, Tabs, Toast, Sheet, Label

## Mobile-First Critical Rules

1. Design at 375px base viewport
2. Touch targets minimum 44x44px
3. Bottom navigation consideration for store pages
4. Cart sidebar becomes full-screen drawer on mobile
5. Product grid: 2 columns mobile, 3 tablet, 4 desktop
6. Checkout: single column mobile, two-column desktop
